## 1. Introduction

This project implements a **Golden Signal Dashboard** that visualizes **latency**, **traffic**, **errors**, and **saturation** for multiple mock microservices. The goal is to connect DevOps themes—observability, containerization, automated testing, and configuration management—with a runnable artifact suitable for academic evaluation.

## 2. Problem and approach

**Problem:** Operators need a concise view of service health. Raw metric dumps are hard to interpret quickly.

**Approach:** Import time-series rows from `data/sample_metrics.csv`, allow filtering by **service** and **time slice**, plot each golden signal, and compute **Healthy / Warning / Critical** states using documented thresholds.

## 3. Implementation summary

- **Application:** `app.py` (Streamlit + Pandas + Plotly).
- **Data:** Synthetic CSV generated by `utils/generate_sample_data.py`.
- **Quality gate:** `tests/test_metrics_file.py` validates schema, timestamps, and health helpers.
- **DevOps:** `Dockerfile` for portable runs; `.github/workflows/ci.yml` for CI; Puppet samples under `puppet/`.

## 4. Puppet / Monitoring alignment

Monitoring is represented by the **dashboard** and **CSV metrics**. Puppet demonstrates **configuration management** by templating a small file that records the app bind host/port for operators or proxy configuration (`puppet/manifests/init.pp`, `puppet/templates/dashboard.conf.epp`).

## 5. Testing and reproducibility

Tests assert that the metrics file exists, columns match expectations, and `app.load_metrics()` returns data. CI runs the same tests on Ubuntu runners.

## 6. Limitations (honest scope)

- Metrics are **static CSV**, not live Prometheus scrape.
- No login or role-based access control.
- Kubernetes and multi-environment promotion are **out of scope** for this submission.

## 7. Conclusion

The repository delivers a consistent, small, end-to-end DevOps sample: versioned code, documented architecture, automated tests, Docker packaging, CI, and Puppet templates—centered on the four golden signals.
